# INSTRUKSI KERJA SEKUENSIAL

## TAHAP 1: ANALISIS UI & PERENCANAAN (Mengacu pada 'UI-Analysis-Ratmin.md')
Lakukan langkah-langkah berikut berdasarkan gambar yang diunggah:

1. Analisa Elemen: Deteksi field form, dropdown, tombol (Save to Draft, Submit), tabel, dan ikon aksi.
2. Konfirmasi Aktif: Jika ada elemen ambigu (misal dropdown auto-complete vs static, datepicker custom vs native), Anda WAJIB berhenti di tahap ini, bertanya kepada saya, dan memberikan panduan cara saya mengambil informasi teknisnya (via Inspect Element). JANGAN membuat asumsi script jika elemen ambigu.
3. Rancang Data Dummy: Buat struktur JSON data dummy yang realistis dan relevan untuk form tersebut untuk disimpan di file fixture.
4. Rancang Flow Skenario:
   - Ikuti aturan khusus "Input/Create Data Baru": Isi Form -> Klik 'Save to Draft' -> Verifikasi Assertion Toast Pesan Sukses menyimpan Draft -> Klik Submit -> Validasi redirect ke URL List -> Validasi data muncul di tabel dengan ikon aksi yang sesuai Role User (tambah untuk Maker, View untuk Reviewer).
   - Ikuti aturan khusus "Approval" (jika ada tombol Approve): Hanya buat skenario 'Approve'.
   - Konfirmasi skenario "View File" (ikon mata) jika terdeteksi; secara default di-skip.
   - Rancang skenario negatif sederhana (mandatory field kosong, format salah, dll.).

## TAHAP 2: GENERATE SCRIPT (Mengacu pada 'Automation-Ratmin-Data-Produced.md')
Setelah Tahap 1 selesai (dan elemen ambigu dikonfirmasi), generate-lah full code untuk asset berikut. Pastikan mematuhi semua HARD LIMITS (No Side Navigation, No Fixed Wait, No Fragile Selector, No Hardcode Data).

1. File Fixture: `cypress/fixtures/[NAMA_MENU].json`.
2. File Page Object (POM): `cypress/pages/[NAMA_MENU]-page.js`.
3. File Script Test: Sesuai parameter 'GENERATE SKENARIO', buat file di `cypress/e2e/Menu/[NAMA_MENU]/` dengan nama file `[NAMA_MENU]-pos.cy.js` dan/atau `[NAMA_MENU]-neg.cy.js`.
